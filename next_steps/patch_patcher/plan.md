# Patcher Orchestrator Implementation Plan

## Objective
Implement the Patcher Sandbox Validation Loop. The Patcher receives the context pack generated by the Planner Orchestrator, generates a unified diff, applies it to an isolated Sandbox container, and verifies correctness by running tests and linting.

## 1. Patcher Adapter Handoff
**File Target**: `autobot_vscode/local_orchestrator/patcher_orchestrator.py` (New File)
- **Action**: Create `run_patcher()` which takes the `assemble_patcher_input()` context dictionary.
- **Action**: Format the prompt into a strict instruction for generating a unified code diff (`--- a/`, `+++ b/`).
- **Action**: Call the `HF_TGI_ENDPOINT` with `HF_PATCHER_ADAPTER` to generate the code diff.

## 2. Sandbox API Interaction
**File Target**: `autobot_vscode/local_orchestrator/patcher_orchestrator.py`
- **Action**: Implement `validate_patch_in_sandbox(diff: str, test_files: list[str]) -> dict`.
- **Action**: Make a synchronous or asynchronous HTTP POST request to the local sandbox container (`http://localhost:5001/run_tests`).
- **Action**: Parse the response: `{ "passed": bool, "returncode": int, "output": str, "duration_s": float }`.

## 3. Linting Checks
**File Target**: `autobot_vscode/local_orchestrator/patcher_orchestrator.py`
- **Action**: Before sending to the sandbox, perform a lightweight local validation on the generated string.
- **Action**: Verify the diff format is strictly valid (starts with `---` and `+++`, contains `@@`). Reject hallucinated git commands or markdown conversational wrapping.

## 4. Patcher Retry Loop (State Machine prep)
**File Target**: `autobot_vscode/local_orchestrator/patcher_orchestrator.py`
- **Action**: Create a simple `while` loop (or LangGraph node structure if strictly needed, though a while loop is preferred for simplicity unless complex branching is required) to allow the Patcher up to 3 attempts.
- **Action**: If `sandbox` returns `passed=False`, extract the traceback from `output` and append it as `ERROR FEEDBACK` to the Patcher's next prompt.

## 5. Wiring into `app.py`
**File Target**: `autobot_vscode/local_orchestrator/app.py`
- **Action**: Modify the `approve_plan` (or create an `execute_patch`) command.
- **Action**: Chain `assemble_patcher_input()` -> `run_patcher()` -> `validate_patch_in_sandbox()`.
- **Action**: Return the final diff and validation status to the VS Code UI.
